# BUG 004: Heading Section Numbers Duplicated

## Symptom

Every numbered heading in the document shows the number twice:

```
1    1Scope
2    3    Terms and definitions
3.2    4.2Physical and chemical characteristics
3.2.1    4.2.1
A.1    A.1Principle
B.3.1    B.3.1Deionized water,
```

The first number is generated by Word's auto-numbering (defined by the
`Heading1`, `Heading2`, `a2` styles via `<w:numPr>`); the second is the
number that the adapter wrote into the paragraph text.

## Root Cause

The ISO DIS template defines auto-numbering for every heading style:

```xml
<w:style w:styleId="Heading1">
  <w:pPr>
    <w:numPr><w:numId w:val="4"/></w:numPr>
    ...
  </w:pPr>
</w:style>
```

So `<w:p><w:pPr><w:pStyle w:val="Heading1"/></w:pPr><w:r><w:t>Scope</w:t></w:r></w:p>`
already renders as "1 Scope" via the linked numbering definition.

But the adapter is extracting the section number from the presentation
XML's `<fmt-title>` and writing it as text runs before the title text:

```xml
<w:p>
  <w:pPr><w:pStyle w:val="Heading1"/></w:pPr>
  <w:r><w:t>1</w:t></w:r>      <!-- adapter-written number -->
  <w:r><w:t>Scope</w:t></w:r>   <!-- title text only -->
</w:p>
```

Word renders: auto-number "1" + text "1Scope" = "1 1Scope".

## Evidence

```xml
<w:p>
  <w:pPr><w:pStyle w:val="Heading1"/></w:pPr>
  <w:bookmarkStart w:id="1" w:name="_baa7c358-..."/>
  <w:bookmarkEnd w:id="1"/>
  <w:r><w:t>1</w:t></w:r>
  <w:r><w:t>Scope</w:t></w:r>
</w:p>
```

The source presentation XML has the number inside `<fmt-title>` as:

```xml
<fmt-title depth="1">
  <span class="..."><semx element="autonum">1</semx></span>
  <span class="fmt-caption-delim"><tab/></span>
  <semx element="title">Scope</semx>
</fmt-title>
```

## Source of Bug

`lib/isodoc/iso/docx/inline.rb` `render_heading_ordered` skips
`fmt-caption-delim` spans (the tab separators) but does NOT skip the
`<semx element="autonum">` element that carries the section number.
The autonum text is rendered as a normal run.

## Fix

Two options:

**Option A (preferred): strip autonum text from headings**

When rendering a heading whose style has auto-numbering (Heading1-9,
a1-a9, ANNEX), skip any `<semx element="autonum">` content. The
numbering is provided by the style's `numPr`.

The cleanest place is in `render_heading_ordered` — filter out semx
elements whose `element` attribute is `autonum`, plus filter out
caption-delim spans (already done).

**Option B: use fmt-caption-label wrapper**

The autonum is wrapped in `<span class="fmt-caption-label">`. Skip
spans with this class when rendering headings.

## Files to Change

- `lib/isodoc/iso/docx/inline.rb` — extend `render_heading_ordered` to
  skip autonum carriers (both `fmt-caption-label` spans and
  `<semx element="autonum">` children)

## Verification

After the fix:

```xml
<w:p>
  <w:pPr><w:pStyle w:val="Heading1"/></w:pPr>
  <w:r><w:t>Scope</w:t></w:r>   <!-- no number text -->
</w:p>
```

Word renders: "1 Scope" via auto-numbering.
