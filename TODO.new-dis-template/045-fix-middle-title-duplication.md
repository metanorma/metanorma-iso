# TODO 045: Fix Middle Title Duplication

## Status: DONE

## What

The middle title section has TWO `zzSTDTitle` paragraphs (duplicate) plus `MainTitle1` and `MainTitle2`. The reference has only ONE `zzSTDTitle` paragraph.

## Why

### Reference (rice.docx)
```
  72: zzSTDTitle  | Cereals and pulses — Specifications and test methods — Part 1: Rice
```

### Our Output
```
  44: MainTitle1  | Cereals and pulses — Specifications and test methods
  45: MainTitle2  | Part 1: Rice
  46: zzSTDTitle  | Cereals and pulses — Specifications and test methods — Part 1: Rice
  47: zzSTDTitle  | Cereals and pulses — Specifications and test methods — Part 1:Rice
```

### Key Issues
1. **Duplicate zzSTDTitle**: Lines 46 and 47 are near-identical (one has " — " before "Part", other has " —" — spacing differs)
2. **MainTitle1/MainTitle2 not in reference**: The reference only uses `zzSTDTitle`
3. The reference `rice.docx` is the OLD isodoc output — it uses only `zzSTDTitle`

### Root Cause

The `render_middle_title` method creates THREE paragraphs (MainTitle1, MainTitle2, zzSTDTitle). The second zzSTDTitle comes from the combined title with `all_parts.join(" — ")`. But the title_part already includes "Part 1" prefix, so the combined title includes it too.

Looking at the code:
```ruby
all_parts = [title_intro, title_main].compact  # ["Cereals and pulses", "Specifications and test methods"]
part_text = "Part 1: Rice"  # title_part_prefix + title_part
all_parts << part_text
# all_parts = ["Cereals and pulses", "Specifications and test methods", "Part 1: Rice"]
para << all_parts.join(" — ")
# Result: "Cereals and pulses — Specifications and test methods — Part 1: Rice"
```

But there's also a SECOND zzSTDTitle being created. This might be from another code path that also renders a title.

## Architecture

The reference rice.docx uses the OLD isodoc template which only has `zzSTDTitle`. The NEW DIS template (from 15926-100.docx) uses `MainTitle1` + `MainTitle2` + `zzSTDTitle`. Since we're using the new DIS template, we should match the DIS template format.

However, the actual reference we're comparing against (rice.docx) was generated by the OLD pipeline and only has `zzSTDTitle`. We need to determine which format to match.

Decision: Match the DIS template format (MainTitle1 + MainTitle2 + zzSTDTitle) but remove the DUPLICATE zzSTDTitle. The duplicate is the real bug.

Check if the duplicate comes from `render_middle_title` creating the combined title twice, or from another code path also emitting a title paragraph.
