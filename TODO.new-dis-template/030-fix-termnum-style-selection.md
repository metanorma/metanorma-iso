# TODO 030: Fix TermNum Style Selection — Auto-Numbering vs Manual Text

## Status: DONE

## What

The adapter uses `TermNum2` (based on `p2` → `Heading2`) for term numbers, which has auto-numbering. But it also outputs the number as text ("3.1"), causing duplication. The alternative is to use `TermNum` (based on `BaseText`, no auto-numbering) with manual text.

## Why

### Style Hierarchy in Our Template

| Style | Based On | Auto-Numbers? | Use Case |
|-------|----------|---------------|----------|
| `TermNum` | `BaseText` | No | Generic term number (needs manual text) |
| `TermNum2` | `p2` → `Heading2` | Yes (`%1.%2`) | Terms under Heading1 clause |
| `TermNum3` | `p3` → `Heading3` | Yes (`%1.%2.%3`) | Terms under Heading2 clause |

### DIS Template Reference

The DIS template uses `TermNum` (no auto-numbering) with manual "3.1" text. This is simpler and avoids numbering conflicts.

### Our Template

Our template has both `TermNum` and `TermNum2`–`TermNum6`. The `TermNum2`–`TermNum6` styles inherit from heading styles and auto-number. The adapter currently uses `TermNum2` and outputs manual text.

### Options

**Option A: Use `TermNum` with manual text** (matches DIS template reference)
- Simpler, more predictable
- No auto-numbering conflicts
- Adapter controls the exact number text

**Option B: Use `TermNum2` without manual text** (uses auto-numbering)
- Numbers generated by numbering definition
- Must ensure term section is at the right heading level
- More fragile — depends on numbering state

### Recommendation

Use **Option A** (`TermNum` with manual text). It's what the DIS template does, and it's more robust since the adapter controls the numbering text directly from the document model.

## Architecture

In `render_term_number`, use `TermNum` style instead of `TermNum2`. Keep the manual number text output.
