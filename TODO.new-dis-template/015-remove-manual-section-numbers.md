# TODO 015: Remove Manual Section Numbers — Auto-Numbering Duplication

## Status: DONE

## What

All heading paragraphs (Heading1–Heading3, ANNEX, a2–a3, TermNum2) have manual section numbers as text AND auto-numbering from the style's `numPr`. This produces duplicated numbers like "1 1 Scope" or "A.1 A.1 Principle".

## Why

The template's numbering definitions already auto-generate section numbers:

| Style | numId | Format | Auto-Text |
|-------|-------|--------|-----------|
| Heading1 | 4 (abstractNum 3) | `%1` | "1" |
| Heading2 | 4 (abstractNum 3, level 1) | `%1.%2` | "4.1" |
| Heading3 | 4 (abstractNum 3, level 2) | `%1.%2.%3` | "4.2.1" |
| ANNEX | 7 (abstractNum 6) | `Annex %1` (upperLetter) | "Annex A" |
| a2 | 7 (abstractNum 6, level 1) | `%1.%2` | "A.1" |
| TermNum2 | inherits from p2 → Heading2 | `%1.%2` | "3.1" |

The adapter currently outputs the number as text (e.g., `<w:t>1</w:t><w:t>Scope</w:t>`) AND the style auto-numbers. This gives:

```
Actual:  "1 1 Scope"     (auto "1" + text "1" + "Scope")
Expected: "1 Scope"       (auto "1" + text "Scope")
```

### Evidence from DIS Template Reference

In the DIS template, Heading1 paragraphs contain ONLY the title text:
```xml
<w:pStyle w:val="Heading1"/>
<w:t>Scope</w:t>   <!-- No "1" prefix -->
```

The numbering "1" is auto-generated by the numPr. Similarly, ANNEX paragraphs contain only the type and title — no "Annex A" text.

## Architecture

### Heading Numbers

Remove the manual number text from heading paragraphs. The adapter's `render_clause_title` (or equivalent) should output only the title text, not the section number.

**Current behavior** (adapter.rb):
```ruby
# Outputs "1" + tab + "Scope" as text runs
para << section_number
para << "\t"
para << title_text
```

**Required behavior**:
```ruby
# Outputs only "Scope" — the numbering is auto-generated
para << title_text
```

### Annex Numbers

Remove "Annex A" from the ANNEX paragraph text. The auto-numbering generates "Annex A" via the `%1` format.

**Current behavior**: Outputs "Annex A" + br + "(normative)" + br + br + "Title"
**Required behavior**: Outputs br + "(normative)" + br + br + "Title" (the leading br is because the style starts after a page break)

### Term Numbers

**NOTE**: Term numbers are handled separately in TODO 030. If using `TermNum` (no auto-numbering), keep manual text. If using `TermNum2` (auto-numbering), remove manual text.

### Subclause Numbers (a2, a3)

Remove "A.1" prefix from annex subclause paragraphs. The a2/a3 auto-numbering generates it.
