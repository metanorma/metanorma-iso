# TODO 056: Fix Deprecated Term Rendering — Wrong Style

## Status: TODO

## What

Deprecated terms ("DEPRECATED: cargo rice") use `TermsAdmitted` style instead of `DeprecatedTerms` style. The reference uses `DeprecatedTerms`.

## Why

### Reference (rice.docx)
```
 102: Terms            | husked rice
 103: DeprecatedTerms  | DEPRECATED: cargo rice
```

### Our Output
```
  73: Terms0          | husked rice
  74: TermsAdmitted   | DEPRECATED: cargo rice
```

### Root Cause

In the style mapping:
```yaml
deprecated_term: TermsAdmitted
```

Should be:
```yaml
deprecated_term: DeprecatedTerms
```

But wait — the template may not have a `DeprecatedTerms` style. The reference rice.docx was generated by the OLD isodoc template which had `DeprecatedTerms`. Our new DIS template might not have this style.

Need to check: does the DIS template have `DeprecatedTerms`? If not, `TermsAdmitted` is the correct fallback.

Also note: the deprecated term is rendered with "DEPRECATED: " prefix but also as `TermsAdmitted` style, which visually makes it look like an admitted term.

## Architecture

1. Check if `DeprecatedTerms` style exists in the DIS template
2. If yes, update style mapping
3. If no, keep `TermsAdmitted` as the best available style

## Files

- `data/iso-dis/style_mapping.yml` — `deprecated_term` mapping
- `data/iso-dis/template.docx` — check for `DeprecatedTerms` style
