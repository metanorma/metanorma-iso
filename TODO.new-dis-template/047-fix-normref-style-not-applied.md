# TODO 047: Fix Normative Reference Style — normref Not Applied

## Status: DONE

## What

Normative reference entries render with `(none)` style instead of `normref` style. The `bib_item_style` method should return `normref` when `@context.in_normative` is true, but it's returning nil (falling through to no style).

## Why

### Reference (rice.docx)
```
  79: normref | ISO 712:2009[FN1]) , Cereals and cereal products...
  80: normref | ISO 6646:2011, Rice — Determination of the potential milling yield...
```

### Our Output
```
  54: (none) | ISO 712:2009[FN1][BR], Cereals and cereal products...
  55: (none) | ISO 6646:2011, Rice — Determination of the potential milling yield...
```

### Root Cause Analysis

The `bib_item_style` method was updated to:
```ruby
def bib_item_style
  if @context.in_normative
    @resolver.paragraph_style(:normref) || @resolver.paragraph_style(:body_text)
  else
    @resolver.paragraph_style(:biblio_entry)
  end
end
```

But `@resolver.paragraph_style(:normref)` returns `nil` if `:normref` is not in the style mapping. Let me check:

The style_mapping.yml was updated to include:
```yaml
normref: normref
```

But the `DocxStyleMapping.paragraph_style` method may not find it, or the style `normref` may not exist in the template DOCX.

Actually, checking the template styles: the template may not have a `normref` style. The reference DOCX uses `normref` but it was generated by the OLD isodoc pipeline which used a different template. Our new DIS template may not have this style.

Need to verify:
1. Does the template have a `normref` style?
2. Is the style mapping loading correctly?

If the template doesn't have `normref`, we need to either add it to the template or fall back to `BodyText`.

## Architecture

1. Check if `normref` style exists in the template DOCX
2. If not, either add it or use `BodyText` as fallback
3. Debug why `@context.in_normative` might be false when rendering normative refs
