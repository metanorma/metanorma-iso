# Deep Audit: rice_fixed13.docx vs rice_fixed13-repaired.docx

## Summary

Every XML part in our DOCX output differs from Word's repaired output.
The differences fall into 4 categories. **Category 1 is the root cause
of "unreadable content."**

---

## Category 1: mc:Ignorable (ROOT CAUSE)

`compute_ignorable` in `helpers.rb` removes "used" namespace prefixes
from mc:Ignorable. This is wrong — Word requires ALL declared extension
prefixes to be listed in mc:Ignorable, regardless of whether they are
used in that part.

**Evidence**: Every part using `compute_ignorable` has a MISMATCH with
the repaired output. Every part using direct assignment (`EXTENSION_PREFIXES`
or `FULL_IGNORABLE`) MATCHES the repaired output (same set, different order).

| Part | Method | Missing from ours |
|---|---|---|
| document.xml | `compute_ignorable(FULL_IGNORABLE, %w[w14])` | w14 |
| footnotes.xml | `compute_ignorable(FULL_IGNORABLE, %w[w14])` | w14 |
| endnotes.xml | `compute_ignorable(FULL_IGNORABLE, %w[w14])` | w14 |
| header1.xml | `compute_ignorable(FULL_IGNORABLE, %w[w14])` | w14 |
| footer1.xml | `compute_ignorable(FULL_IGNORABLE, %w[w14])` | w14 |
| numbering.xml | `compute_ignorable(FULL_IGNORABLE, %w[w15 w16cid])` | w15, w16cid |
| settings.xml | `compute_ignorable(EXTENSION_PREFIXES, %w[w14 w15])` | w14, w15 |
| fontTable.xml | direct `EXTENSION_PREFIXES` | none (MATCH) |
| styles.xml | direct `EXTENSION_PREFIXES` | none (MATCH) |
| webSettings.xml | direct `EXTENSION_PREFIXES` | none (MATCH) |

**Fix in uniword**: Remove `compute_ignorable`. Replace all calls with
direct assignment of the full prefix list:

- FULL_IGNORABLE for: document, footnotes, endnotes, headers, footers,
  numbering (body-level parts)
- EXTENSION_PREFIXES for: settings, fontTable, styles, webSettings
  (document-level parts)

**Files to change**:
- `lib/uniword/docx/reconciler/helpers.rb` — delete `compute_ignorable`
- `lib/uniword/docx/reconciler/parts.rb` — settings (line ~70),
  numbering (line ~147), document_body (line ~273)
- `lib/uniword/docx/reconciler/notes.rb` — footnotes/endnotes (line ~80)
- `lib/uniword/docx/reconciler/body.rb` — headers/footers (line ~58)
- `spec/uniword/docx/mc_ignorable_spec.rb` — update expectations to
  match full prefix lists (no prefix removal)

## Category 2: Namespace declarations

### numbering.xml: missing standard Office namespaces

Ours declares 12 namespace prefixes (extension + mc + w).
Repaired declares all 35 standard Office namespaces.

The namespaces actually used in numbering.xml are: mc, w, w15, w16cid.
Both files declare all of these. The extra 23 namespaces in the repaired
version are declared-but-unused — Word adds them for consistency.

**Impact**: Likely cosmetic. Not a schema violation since declared-but-unused
namespaces are valid XML. But Word's normalization suggests it prefers
the full set.

### styles.xml: extra wp14 namespace declaration

Ours declares wp14 (Word 2010 Drawing) namespace. Repaired does not.
wp14 is not used in styles.xml content.

**Impact**: Cosmetic. Unused namespace declarations don't cause errors.

## Category 3: Document content differences

These are content-level differences generated by our builder pipeline
(metanorma-iso adapter). They are not structural errors but represent
divergent content choices.

### document.xml (174KB ours vs 120KB repaired — 54KB difference)

| Element | Ours | Repaired | Notes |
|---|---|---|---|
| `<tbl>` | 2 | 1 | Two adjacent tables merged by Word |
| `<gridCol>` | 10 | 5 | Extra table has 5 grid columns |
| `<b>` (bold) | 5 | 0 | Redundant bold in ANNEX-styled paragraphs |
| `<r>` (runs) | 1285 | 792 | Our runs are more granularly split |
| `<t>` (text) | 1145 | 731 | Corresponding to extra runs |
| `<lastRenderedPageBreak>` | 0 | 16 | Word adds these during save |
| `<p>` (paragraphs) | 418 | 419 | ±1 from table merge |

**Bold runs**: 5 runs with `<w:b/>` inside `<w:rPr>` inside paragraphs
with `ANNEX` style. The ANNEX style definition likely already includes
bold, making these redundant. Word strips them during repair.

**Run splitting**: Our adapter generates ~500 more runs than Word by
splitting text into smaller run units. This is valid OOXML but verbose.
Word consolidates adjacent runs with identical formatting.

**Adjacent tables**: Two tables at body positions 285 and 286 with
identical column structure (5 cols). Word merges them into one 29-row
table during repair. Producing a single table would match Word's
expectation.

### settings.xml (11.7KB vs 10.8KB)

- Zoom: ours `bestFit`/`160%` vs repaired `104%` — different zoom mode
- Rsids: 74 vs 78 — Word adds extra rsid entries during repair
- w14:docId: different hex values (expected — random generation)

### app.xml (1.1KB vs 0.7KB)

- Ours includes `<HeadingPairs>` and `<TitlesOfParts>` with variant
  vectors; repaired doesn't have them
- Statistics values differ (ours: 4 pages/489 words, repaired:
  18 pages/3286 words)

### core.xml

- Revision: ours `5`, repaired `1`
- Modified timestamp: different (expected)
- Content otherwise identical

### theme/theme1.xml (11.5KB vs 8.3KB)

Size difference likely from namespace declaration differences or
formatting normalization.

## Category 4: Cosmetic differences (no impact)

### Attribute ordering on `<w:p>`

- Ours: `w:rsidR`, `w:rsidRDefault`, `w14:paraId`, `w14:textId`
  (w namespace first)
- Repaired: `w14:paraId`, `w14:textId`, `w:rsidR`, `w:rsidRDefault`
  (w14 namespace first)

XML attribute order is not significant per XML spec. No impact.

### Root element namespace declaration order

Each part has the same set of namespace declarations but in different
order. No impact.

### Content type / relationship ordering

`[Content_Types].xml` entries and `.rels` relationships have the same
members in different order with different rId numbering. No structural
impact.

## Fix Priority

1. **Category 1** (mc:Ignorable) — CRITICAL. This is the root cause of
   "unreadable content." Remove `compute_ignorable` and use full prefix
   lists. Fix in 4 reconciler files + update spec.

2. **Category 3** (content) — OPTIONAL. These don't cause errors but
   could be improved for fidelity:
   - Strip redundant bold from ANNEX-styled runs
   - Consolidate adjacent runs with identical formatting
   - Merge adjacent tables with matching structure
   - Remove HeadingPairs/TitlesOfParts from app.xml
   - Fix zoom value handling

3. **Category 2** (namespaces) — LOW PRIORITY. No error impact.

4. **Category 4** (cosmetic) — NO ACTION NEEDED.
